### Главная цель MVP

Собрать Telegram-бота для магазина цветов в Красноярске, где пользователь может:

- выбрать повод и бюджет;
- получить подходящий букет с фото и смыслом;
- оформить доставку;
- запросить консультацию флориста;
- передать заказ в работу курьеру.

### Что сейчас не делаем

- полную аналитику сотрудников;
- авто-распределение заказов между несколькими курьерами;
- версию для ВКонтакте.

## 2. Итог, к которому должны прийти

К концу MVP должны быть готовы:

- стабильный Telegram bot flow по ТЗ;
- подбор и каталог букетов из БД (без mock);
- оформление заказа и отправка данных курьеру;
- заявка на консультацию и отправка данных флористу;
- хранение заказов и заявок в Django;
- базовая статистика в web-интерфейсе (admin);
- согласие на обработку персональных данных;
- готовность к подключению Telegram Payments.

## 3. Локальный запуск

1. Активировать виртуальное окружение:

```powershell
.\.venv\Scripts\Activate
```

2. Установить зависимости:

```powershell
pip install -r requirements.txt
```

3. Миграции:

```powershell
python manage.py makemigrations
python manage.py migrate
```

4. Запуск Django:

```powershell
python manage.py runserver
```

5. Запуск бота:

```powershell
python bot/main.py
```

## 6. Что можно делать параллельно

- Stream A: Этап 2 (модели/миграции/admin).
- Stream B: Этап 3 (переход на БД для букетов).
- Stream C: Этап 5 + Этап 6 (уведомления и консультации).
- Stream D: Этап 7 + Этап 8 (доп. фильтры).
- Stream E: Этап 9 (платежи).
- Stream F: тесты и документация на каждый этап.

## 7. Что тестировать обязательно

1. `/start` и выбор всех вариантов повода.
2. Все варианты бюджета, включая `не важно`.
3. Негативный кейс: букетов нет.
4. Создание заказа с валидацией всех полей.
5. Отправка заявки флористу.
6. Отправка заказа курьеру.
7. Согласие/отказ по ПДн.
8. Оплата (после подключения).

## 8. Минимальные технические соглашения

- Вся бизнес-логика вне handlers (в services/use_cases).
- Один callback = один чёткий контракт.
- Любое изменение модели = миграция + admin + тест.
- Никаких mock-данных в production-ветке.

## 9. Как понять, что MVP готов

MVP готов, если стабильно работает сценарий:

1. Пользователь запускает бота.
2. Выбирает повод/бюджет.
3. Получает релевантный букет из БД.
4. Оформляет заказ с ПДн-согласием.
5. Заказ сохраняется.
6. Курьер получает уведомление.
7. При консультации флорист получает контакт.
8. Владелец видит заказы и базовую статистику.
