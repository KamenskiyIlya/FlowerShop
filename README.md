# FlowerShop

Telegram-бот цветочного магазина на `pyTelegramBotAPI` + backend на `Django`.
Проект закрывает подбор готовых букетов из БД, оформление заказа, консультации флориста и web-интерфейс владельца через Django admin.

## Основные возможности

- Подбор букетов по поводу и бюджету из БД
- Показ карточки букета: фото, смысл, состав, цена
- Каталог с навигацией по букетам
- Оформление заказа с сохранением в БД
- Промокоды и пересчет итоговой суммы заказа
- Сохранение Telegram-пользователя без дублей
- Создание заявки на консультацию с сохранением в БД
- Уведомления сотрудникам (курьеры и флористы) через Telegram
- Кнопка `В главное меню` на ключевых этапах сценариев
- Страница статистики заказов в Django admin
- Страница статистики консультаций в Django admin

## Архитектура проекта

```bash
manage.py

flower_shop/
  settings.py              # Настройки Django
  urls.py                  # Маршруты admin и media

bot/
  main.py                  # Точка входа Telegram-бота
  config/settings.py       # Загрузка env-конфига для бота
  handlers/                # Сценарии бота
  keyboards/               # Inline/reply клавиатуры
  services/                # Бизнес-логика (order, consultation, promo, notifications)

bot_app/
  models.py                # Доменная модель (TgUser, Bouquet, Order, Consultation, Employee, PromoCode)
  admin.py                 # Админка + кастомные страницы статистики
  migrations/              # Django-миграции
  templates/admin/         # Кастомные шаблоны страниц статистики
```

## Данные и сущности

Ключевые модели:

- `TgUser` — Telegram-пользователь, сохраняется по `telegram_id`
- `Bouquet` — букет и его характеристики
- `Order` — заказ, сумма, статус, доставка, назначенный курьер
- `Consultation` — заявка на консультацию и ее жизненный цикл
- `Employee` — сотрудники (флорист, курьер, директор), состояние занятости
- `PromoCode` — промокоды и скидки

## Переменные окружения

Создайте `.env` в корне проекта:

```env
BOT_TOKEN=your_telegram_bot_token
DEBUG=True
```

## Установка и запуск

```bash
pip install -r requirements.txt
python manage.py migrate

# Если проект запускается локально, тогда отдельно запускается админ-панель:
python manage.py runserver
# И в отдельном терминале сам бот:
python bot/main.py
```

## Основной пользовательский процесс

1. `/start`
2. Выбор повода (или ввод своего)
3. Выбор бюджета
4. Получение подборки подходящих букетов
5. `Заказать букет`:
   имя -> адрес -> дата/время -> промокод (опционально) -> сохранение заказа
6. Уведомление курьеров о новом заказе
7. `Консультация`:
   ввод телефона -> сохранение заявки -> уведомление флористов

## Админка и статистика

Доступно в Django admin:

- Справочники и сущности: пользователи, букеты, заказы, консультации, сотрудники, промокоды.
- Возможность добавлять и убирать букеты в базе данных. Если какой-то букет закончился, не обязательно его удалать полностью, достаточно убрать отметку "в наличии":

<img width="1000" alt="image" src="https://github.com/user-attachments/assets/dda86f7b-584c-4645-8ae6-ec9e0c8e3669" />

- Раздел "заказы" позволяет просматривать, а также редактировать в случае необходимости поступившие заказы.
- Раздел "заявки на консультацию" позволяет просматривать новые заявки, обрабатывать их и закрывать.
- Раздел "клиенты в telegram" дает возможность просматривать базу клиентов, которые когда либо обращались к боту. В случае необходимости можно редактировать данные или удалять записи из базы данных.
- Раздел "промокоды" позволяет создавать, а также работать с промокодами. Основные возможности - указать название промокода, размер скидки, активен ли промокод(стандартно активен), а также дату и время до какаго момента промокод активен.

<img width="600" alt="image" src="https://github.com/user-attachments/assets/8ea11ec3-417b-4116-88fc-99ec76ad5bbb" />

- Раздел "сотрудники" позволяет заводить карточки сотрудников, назначать на определенные должности, а также указывать статус сотрудника например "больничный" или "в отпуске" (стандартно статус "работает).
- Отправка уведомлений происходит сотрудникам, которые добавлены в базу данных и которые работают. Соответственно флористам приходят оповещения о новых запросах на консультации, а курьерам оповещения о новых заказах. Если на момент заказа или запроса консультации нет ни одного сотрудника, который мог бы обработать эти запросы, оповещение об этом направляется руководителю.
- Статистика заказов.
